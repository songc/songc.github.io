Hexo 中渲染 MathJax 数学公式的问题

最近，在使用Hexo写博客的过程中，发现Hexo中的markdown文件中的数学公式无法正常的渲染。问题折腾了好久，在网上查找了不少的资料，最后使用hexo-renderer-pandoc这个插件解决了该问题。 结果过程如下:

  1. 首先电脑上要先安装pandoc该软件，安装过程比较简单，根据提示安装就行。
  2. 然后在博客的根目录下打开终端 输入命令npm install hexo-renderer-pandoc --save 安装该插件。
  3. 然后卸载掉hexo的其他markdown的渲染引擎npm uninstall hexo-renderer-marked hexo 默认是使用marked 该渲染引擎。如果有其他的渲染引擎也要一起卸载掉(渲染引擎是 根目录下package.json 中dependencies 下的以hexo-renderer-**开头的依赖库)。(我在只卸载了默认的marked渲染引擎后，发现还是渲染失败，然后我尝试将package.json中hexo-renderer-pandoc移动到了渲染引擎的第一个，然后发现可以成功的渲染MathJax数学公式。个人猜测 hexo是按照package.json中渲染引擎的顺序逐个依次进行渲染的。所以也可以不卸载hexo的其他渲染引擎，在安装pandoc插件后，修改package.json中的渲染引擎的顺序，将pandoc引擎放到第一位就可以了)

其他的一些解决方案:

网上还有一些其他的解决方案:

  1. hexo-math 插件，但是我安装该插件后不能完美的解决问题，有些公式无法正常解析。

  2. hexo-renderer-mathjax,这个插件我也尝试过，但是也有问题。

  3. Next主题是自带Mathjax的功能，默认是关闭的可以在Next主题的配置文件下打开就可以，设置_config.yml

但是自带的解析功能不好用，存在问题。

Markdown无法解析的原因：

markdown中数学公式无法正常显示，是渲染引擎的原因。具体的原因可以看着两片文章解释的比较清楚Hexo下mathjax的转义问题,数学公式和Markdown的结合
